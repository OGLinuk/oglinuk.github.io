Nginx ~ Subdomains and a Blog

In the previous blog we explored serving a Go web application with Nginx on Digital Ocean. This blog is going to cover how to host a static website on a subdomain. One of the things I did not cover in the last blog is how to point to your domain name with Digital Ocean. For more on this please refer to this blog that I found/used originally.

The first thing to do is create another droplet. There might be a way to host the static website on the same droplet, but I have not figured that out yet. Once you have your droplet running and up-to-date, install Nginx via sudo apt install nginx. After you’ve installed Nginx, copy over your static site using an scp or rsync command. If you’re wondering what the difference between the two like I did, you can refer to this stackexchange answer. I copied my static website to the /var/www directory, but you can choose to store it wherever you would like, just make sure you use that same path in the Nginx configuration.

The next thing to do is create a Nginx server block file in /etc/nginx/sites-available. Add the following to the file.

server {
	root /var/www/fourohfournotfound;

	server_name blog.fourohfournotfound.com www.blog.fourohfournotfound.com;
}

Finally, to add a subdomain to point to the droplet. In my case, since my static website is my blog I am adding blog and www.blog as the hostname directing to the droplet with the static website.
